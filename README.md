# Pizza Sales Analysis Project
This project focuses on analyzing retail pizza sales data using MySQL to generate meaningful business insights.

## Project Objectives
1 Clean dataset by identifying and handling NULL values
2 Analyze sales performance by category, customer demographics, and product types
3 Identify top-selling products and highest revenue contributors
4 Study order patterns by date and hour to find peak demand times
5 Calculate metrics such as total sales, quantity sold, and average orders per day

## Tools & Technologies
1 MySQL – data querying and analysis
2 Excel – dataset storage and basic preprocessing
3 GitHub – version control and project documentation

## Dataset Description
1. Customer details
2. Order ID and order date
3. Product name and category
4. Quantity ordered
5. Price per unit
6. Total transaction amount

## Business Insights Generated
1. Identified top-performing pizza categories
2. Determined peak ordering hours and busiest days
3. Found highest revenue-contributing products
4. Analyzed customer ordering patterns
5. Measured overall sales performance

## Conclusion
The Pizza Sales Analysis project successfully transformed raw transactional data into meaningful business insights using MySQL. Through data cleaning, aggregation, and trend analysis, key performance metrics such as total revenue, top-selling products, peak demand hours, and category-wise performance were identified. The analysis revealed important customer ordering patterns and high-revenue contributors, helping understand overall sales behavior.
